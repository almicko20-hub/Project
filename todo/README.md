# My ToDo

Одностраничное веб-приложение для списка задач. Сделано на чистом HTML, CSS и JavaScript (без сборщиков и фреймворков).

## Как запустить

### Вариант A — Docker (рекомендуется)

В проекте есть готовый `Dockerfile` (статика отдаётся встроенным HTTP-сервером Python, `python -m http.server`, без nginx и docker-compose).

```bash
# собрать образ
docker build -t my-todo .

# запустить контейнер
docker run -d -p 8080:80 --name my-todo my-todo
```

Откройте в браузере: **http://localhost:8080**

Остановить и удалить контейнер:
```bash
docker stop my-todo && docker rm my-todo
```

### Вариант B — без Docker

Приложение не требует сервера и сборки — достаточно открыть `index.html` в браузере, либо поднять любой статический сервер:
```bash
npx serve .
```

## Технологии

- **HTML5** — семантическая адаптивная разметка.
- **CSS3** — собственная стилизация в духе «бумажного блокнота» (без фреймворков), адаптив через медиа-запросы, поддержка `prefers-reduced-motion`.
- **Vanilla JavaScript (ES6+)** — вся логика без сторонних UI-фреймворков.
- **DOMPurify** — санитизация пользовательского ввода перед вставкой в DOM (защита от XSS), подключена локально (`js/vendor/purify.min.js`), без обращения к внешним CDN.
- **localStorage** — хранение задач между перезагрузками страницы.

## Функциональность (Раздел 1 ТЗ)

- Добавление новой задачи через форму.
- Отметка задачи как выполненной (чекбокс).
- Удаление задачи.
- Фильтрация: «Все» / «Активные» / «Выполненные».
- **Сортировка** списка (выпадающий список рядом с фильтрами):
  - «Сначала новые» — по дате добавления, от новых к старым (по умолчанию);
  - «Сначала старые» — по дате добавления, от старых к новым;
  - «По алфавиту (А–Я)» — по тексту задачи;
  - «Сначала невыполненные» — невыполненные задачи выше выполненных.

  Фильтр и сортировка применяются независимо и работают вместе. Выбранный режим сортировки запоминается в `localStorage` и восстанавливается при следующем открытии страницы.
- Адаптивный интерфейс: заголовок → форма добавления → фильтры/сортировка → список задач, перестраивается на мобильных экранах (блок фильтров и сортировки переходит в колонку).

## Оптимизация (Раздел 2 ТЗ)

- Скрипты подключены с атрибутом `defer` (`index.html`), не блокируют рендер страницы.
- Готовые минифицированные версии стилей и скриптов: `css/style.min.css`, `js/app.min.js` (исходники — `css/style.css`, `js/app.js`).
- Список задач хранится в `localStorage`, чтобы не терялся при обновлении страницы.

## Безопасность (Раздел 3 ТЗ)

- Пользовательский ввод обрабатывается через `DOMPurify.sanitize()` перед сохранением и отображением.
- Текст задачи выводится через `textContent`, а не `innerHTML`, чтобы избежать XSS.
- В `index.html` задан мета-тег `Content-Security-Policy`, запрещающий инлайн-скрипты и загрузку ресурсов с посторонних доменов. Директива `style-src` не содержит `'unsafe-inline'`, так как в проекте нет инлайн-стилей — все стили подключаются файлом. Дополнительно задан `form-action 'self'`.
